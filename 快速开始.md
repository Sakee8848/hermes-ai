### 4️⃣ 风险分析 (Risk Analysis)

**访问**: http://localhost:3000/dashboard/risk

**功能**:
- 实时风险监控
- AI智能检测
- 合规性扫描
- 制裁名单比对

**风险等级**:
- 🔴 高 (High): 需立即处理
- 🟡 中 (Medium): 需关注
- 🔵 低 (Low): 常规监控

**操作流程**:
```
1. 查看风险概览
   - 总警报数
   - 今日已解决
   - AI准确率

2. 筛选风险警报
   - 按严重程度筛选
   - 按类别筛选（合规/文档/路线/财务）

3. 处理风险警报
   - 点击"View Details"查看详情
   - 点击"Acknowledge"确认
   - 点击"Mark Resolved"标记已解决
```

**AI置信度**:
- 每个警报都有AI置信度评分
- 评分越高，准确性越高
- 建议优先处理高置信度警报

### 5️⃣ 运单追踪 (Shipment Tracking)

**访问**: http://localhost:3000/dashboard/shipments

**功能**:
- 实时运单追踪
- 航线可视化
- 延误预警
- 送达确认

**运单状态**:
- ⚓ 在港 (At Port)
- 🚢 在途 (In Transit)
- ⚠️ 延误 (Delayed)
- ✅ 已送达 (Delivered)

**操作流程**:
```
1. 查看运单列表
   - 提单号和集装箱号
   - 船舶和航次信息
   - 起运港和目的港

2. 监控运输进度
   - 查看进度条
   - 查看当前位置（动画标记）
   - 查看ETD/ETA时间

3. 处理延误运单
   - 识别延误标记
   - 查看延误原因
   - 采取相应措施
```

**进度可视化**:
- 蓝色进度条: 正常在途
- 黄色进度条: 延误
- 绿色进度条: 已送达
- 动画标记: 当前位置

## ❓ 常见问题

### Q1: 如何启动开发服务器？
```bash
cd /Users/tonyyu/Documents/物流行业/Hermes-AI/frontend
npm run dev
```

### Q2: 端口被占用怎么办？
```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或使用其他端口
PORT=3001 npm run dev
```

### Q3: 页面显示空白？
**解决方案**:
1. 清除浏览器缓存
2. 检查控制台错误
3. 重启开发服务器
4. 检查Node.js版本

### Q4: 如何查看所有可用命令？
```bash
npm run
```

### Q5: 如何构建生产版本？
```bash
npm run build
npm run start
```

### Q6: 移动端如何访问？
1. 确保设备在同一局域网
2. 访问: http://192.168.1.48:3000
3. 或使用手机浏览器扫描二维码

## 🔧 故障排除

### 问题1: npm install 失败
```bash
# 清除缓存
npm cache clean --force

# 删除node_modules
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 问题2: 编译错误
```bash
# 检查TypeScript错误
npm run lint

# 清除Next.js缓存
rm -rf .next
npm run dev
```

### 问题3: 样式不生效
```bash
# 重启Tailwind编译
# Ctrl+C 停止服务器
npm run dev
```
